# FourSight (DORA Tracker for GitHub) 要求仕様書

## 1. プロダクト概要

GitHubリポジトリのデータから「Four Keys（DORAメトリクス）」を自動算出し、ブラウザ上の専用ダッシュボードで可視化する開発者向けブラウザ拡張機能。

## 2. 動作環境
-   **対象ブラウザ:** Google Chrome, Microsoft Edge（Manifest V3対応。ExtensionHubやEdgeアドオンストアでのスムーズな公開を想定した設計）
-   **対象ユーザー:** GitHubを利用するソフトウェアエンジニア、EM（エンジニアリングマネージャー）、スクラムマスター
-   **認証方式:** BYOI（Bring Your Own Identity）方式。ユーザー個人の Personal Access Token (PAT) を使用。

## 3. システム構成（Manifest V3 アーキテクチャ）

-   **Service Worker (`background.js`):**
    -   拡張機能アイコンのクリックイベントの検知。
    -   トークンの有無を判定し、ダッシュボード画面（新規タブ）へのルーティング（`chrome.tabs.create`）。
-   **Dashboard Page (`dashboard.html / Dashboard.vue`):**
    -   独立したタブとして開かれるメインのUI（SPA構成）。
    -   PrimeVueコンポーネントを用いたリッチなUIと、Chart.jsを用いたメトリクスの可視化（折れ線・棒グラフの切り替え対応）。
    -   **設定モーダル（旧Options Page）**: ダッシュボード内に統合され、PATの設定や言語切り替え、障害判定ラベルの設定をシームレスに行う。
-   **Storage (`chrome.storage.local`):**
    -   PATのセキュアな保存。  
    -   カスタマイズ設定（表示言語や障害判定ラベルなど）の保存。

## 4. コア機能要件

### 4.1. 初期設定・認証フロー
1.  ユーザーが拡張機能アイコンをクリックし、ダッシュボード画面を開く。
2.  `chrome.storage.local` にPATが存在しない場合、自動的に設定モーダルが開く。
3.  モーダル内には GitHub のトークン発行ページへの案内と、必要なスコープ（`repo`, `read:org`）を記載。
4.  PATを入力・保存すると、API通信が可能になる。

### 4.2. メトリクス算出ロジック（GitHub GraphQL API利用）

設定したルールに基づき、以下のデータを取得・算出する。

| 指標 | 算出ルール |
| --- | --- |
| **デプロイ頻度 (DF)** | 指定ブランチ（デフォルト: `main`）への Pull Request マージ回数 |
| **変更リードタイム (LTFC)** | 対象PRの「First Commit 時刻」から「マージ時刻」までの時間差（日数） |
| **変更障害率 (CFR)** | デプロイ（PRマージ）後に作成された、指定ラベル（デフォルト: `bug`, `incident`）が付与された Issue の数 ÷ デプロイ回数 (%) |
| **サービス復元時間 (TRRS)** | 上記の障害 Issue の「作成時刻」から「Close 時刻」までの時間差（時間） |

### 4.3. ダッシュボード画面機能

-   **リポジトリ検索とオートコンプリート:** PATの権限でアクセス可能なリポジトリをインクリメンタルサーチで検索できる。
-   **期間・ブランチ指定:** 日付範囲の選択とターゲットブランチの指定が可能。
-   **サマリーカード:** Four Keys（DF, LTFC, CFR, TRRS）それぞれの値を算出・表示。
-   **アクティビティ推移グラフ:** 日別のデプロイ回数と障害発生数のタイムライン・チャート。折れ線グラフと棒グラフの切り替えが可能。
-   **多言語対応 (i18n):** 外部ライブラリに依存せず、設定モーダルから日本語と英語のUI表示をリアルタイムに切り替え可能。
-   **デモモード:** URLに `?demo=true` を付与することで、ストア公開キャプチャ用のダミーデータを自動生成して表示する機能を搭載。

## 5. CI/CD および公開プロセス

-   **自動タグ作成:** `main` ブランチへのプッシュ時に、`manifest.json` に記載されたバージョン番号を読み取り、GitHub Actionsが自動的にGitタグを作成してプッシュする。
-   **Chromeウェブストア自動公開:** GitHub Releaseが作成・公開された際に、拡張機能をビルドし、`chrome-webstore-upload-cli` を用いて自動的にChromeウェブストアへアップロード・審査提出を行うワークフローを構築。
