# FourSight (DORA Tracker for GitHub) 要求仕様書

## 1\. プロダクト概要

GitHubリポジトリのデータから「Four Keys（DORAメトリクス）」を自動算出し、ブラウザ上の専用ダッシュボードで可視化する開発者向けブラウザ拡張機能。

## 2\. 動作環境
-   **対象ブラウザ:** Google Chrome, Microsoft Edge（Manifest V3対応。ExtensionHubやEdgeアドオンストアでのスムーズな公開を想定した設計）
-   **対象ユーザー:** GitHubを利用するソフトウェアエンジニア、EM（エンジニアリングマネージャー）、スクラムマスター
-   **認証方式:** BYOI（Bring Your Own Identity）方式。ユーザー個人の Personal Access Token (PAT) を使用。

## 3\. システム構成（Manifest V3 アーキテクチャ）

-   **Service Worker (`background.js`):**
    -   拡張機能アイコンのクリックイベントの検知。
    -   トークンの有無を判定し、設定画面またはダッシュボード画面（新規タブ）へのルーティング（`chrome.tabs.create`）。
-   **Dashboard Page (`dashboard.html / js`):**
    -   独立したタブとして開かれるメインのUI。
    -   グラフ描画ライブラリ（Chart.js, Recharts等）を用いたメトリクスの可視化。 
-   **Options Page (`options.html / js`):**
    -   PATの設定・保存機能。
    -   リポジトリごとの計測ルール（運用定義）のカスタマイズ画面。
        
-   **Storage (`chrome.storage.local`):**
    -   PATのセキュアな保存。  
    -   カスタマイズ設定の保存。
    -   APIリクエスト削減のための取得データキャッシュ。

## 4\. コア機能要件

### 4.1. 初期設定・認証フロー
1.  ユーザーが拡張機能アイコンをクリックする。
2.  `chrome.storage.local` にPATが存在しない場合、自動的に Options Page が開く。
3.  Options Page には GitHub のトークン発行ページへのリンクと、必要なスコープ（`repo`, `read:org` 等）の案内を記載。
4.  PATを入力・保存すると、Dashboardへのアクセスが可能になる。

### 4.2. メトリクス算出ロジック（GitHub GraphQL API利用）

事前に Options Page で設定したルールに基づき、以下のデータを取得・算出する。

| 指標 | デフォルトの算出ルール（※ユーザー設定で上書き可能） |
| --- | --- |
| **デプロイ頻度 (DF)** | `main` ブランチへの Pull Request マージ回数 |
| **変更リードタイム (LTFC)** | 対象PRの「First Commit 時刻」から「マージ時刻」までの時間差の中央値 |
| **変更障害率 (CFR)** | デプロイ（PRマージ）後に作成された、`bug` または `incident` ラベルが付与された Issue の数 ÷ デプロイ回数 |
| **サービス復元時間 (TRRS)** | 上記の障害 Issue の「作成時刻」から「Close 時刻」までの時間差の中央値 |

#### 4.3. ダッシュボード画面 (Portal & Dashboard Page)

独立したタブで開かれるメイン画面。SPA（Single Page Application）のようなスムーズな遷移を想定。

-   **ポータル・ホーム画面 (未選択状態):**
    -   **リポジトリ検索バー:** PATの権限でアクセス可能なリポジトリをインクリメンタルサーチ（自動補完）で検索できる。
    -   **お気に入り / 最近見たリポジトリ:** よく見るリポジトリをピン留め、または履歴として表示し、ワンクリックでダッシュボードへ遷移できる機能。
-   **メトリクス・ビュー (リポジトリ選択後):**
    -   **ヘッダー:** 選択中のリポジトリ名へのリンク、設定（Options）へのショートカット、期間指定トグル（例: 過去7日、14日、30日）。
    -   **サマリーカード:** Four Keys（DF, LTFC, CFR, TRRS）それぞれの平均値/中央値と、前期比（トレンドの矢印）を表示。
    -   **推移グラフ:** 日別・週別のデプロイ回数と障害発生数のタイムライン・チャート。
    -   **手動リロードボタン:** キャッシュを破棄して最新データをGraphQLで再取得する。
        
#### 4.4. データ・キャッシュ戦略 (`chrome.storage.local`)
-   **最近の検索履歴:** 検索バーでの利便性向上のため保存。
-   **リポジトリ別キャッシュ:** `{ "owner/repo": { metrics: {...}, lastFetched: "timestamp" } }` の形式で保存。一定時間（例: 1時間）はAPIを叩かずキャッシュから即座に描画する。
