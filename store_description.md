### **English Version**

### **Name**
FourSight (DORA Tracker for GitHub)

### **Short Description**
Automatically calculate and visualize Four Keys (DORA metrics) directly from your GitHub repository data.

### **Detailed Description**
[ PURPOSE ]
FourSight is a powerful browser extension that automatically calculates and visualizes the "Four Keys" (DORA metrics) directly from your GitHub repositories. By instantly analyzing your pull requests and issues, it helps engineering teams measure their software delivery performance without setting up complex third-party tools or servers.

[ CORE FEATURES ]
- Automatic DORA Metrics Calculation: Instantly measures Deployment Frequency (DF), Lead Time for Changes (LTFC), Change Failure Rate (CFR), and Time to Restore Service (TRRS).
- Visual Activity Trends: Provides beautiful, interactive charts (switchable between Line and Bar) to track your team's development activity over time.
- Smart Repository Search: Auto-suggests repositories you have access to, making it easy to analyze your own projects.
- Flexible Filtering: Filter metrics by specific date ranges and target branches.

[ WHAT MAKES IT SPECIAL ]
- Zero Server Setup: Everything runs directly in your browser. No external databases, no third-party servers, and no complex CI/CD integrations required.
- Multilingual Interface: Seamlessly switch between English and Japanese directly from the settings.

[ HOW TO USE ]
1. Install the extension and open the FourSight Dashboard.
2. Click the gear icon to set your GitHub Personal Access Token (requires 'repo' and 'read:org' scopes).
3. Search for your repository, select a date range, and click the refresh button to visualize your metrics!

[ PRIVACY FIRST ]
Your data stays yours. FourSight communicates directly and securely with the official GitHub GraphQL API. Your GitHub Personal Access Token is saved locally in your browser's secure storage and is never transmitted to any external third-party servers.

### **Japanese Version**

### **Name**
FourSight (DORA Tracker for GitHub)

### **Short Description**
GitHubリポジトリのデータから「Four Keys（DORAメトリクス）」を自動算出し、ダッシュボードで可視化します。

### **Detailed Description**
【 目的 】
FourSight（フォーサイト）は、GitHubリポジトリのデータから直接「Four Keys（DORAメトリクス）」を自動的に算出し、可視化する強力なブラウザ拡張機能です。複雑な外部ツールやサーバーを構築することなく、Pull RequestやIssueを瞬時に分析し、エンジニアリングチームの開発パフォーマンス測定をサポートします。

【 主な機能 】
・ DORAメトリクスの自動算出: デプロイ頻度 (DF)、変更リードタイム (LTFC)、変更障害率 (CFR)、サービス復元時間 (TRRS) を瞬時に計測します。
・ アクティビティ推移の可視化: 開発アクティビティの推移を、見やすくインタラクティブなグラフ（折れ線・棒グラフ切替可能）で表示します。
・ スマートなリポジトリ検索: 自身がアクセス権を持つリポジトリのみをサジェスト表示し、素早い分析を可能にします。
・ 柔軟なフィルタリング: 任意の集計期間やターゲットブランチを指定してメトリクスを絞り込むことができます。

【 特徴的な機能 】
・ サーバー構築不要: 全ての計算処理はブラウザ上で完結します。外部データベースやサードパーティ製サーバーの契約、複雑なCI/CD連携は一切不要です。
・ 多言語対応: 設定画面から日本語と英語のUIをシームレスに切り替えることができます。

【 使用方法 】
1. 拡張機能をインストールし、FourSightのダッシュボードを開きます。
2. 歯車アイコンをクリックし、GitHub Personal Access Token（repo および read:org スコープが必要）を設定します。
3. 分析したいリポジトリと期間を選択し、更新ボタンを押すだけでメトリクスが可視化されます。

【 プライバシー最優先 】
ユーザーのデータは安全に保護されます。FourSightは公式のGitHub GraphQL APIと直接通信を行います。設定したGitHubトークンはブラウザのローカルストレージにのみ安全に保存され、外部のサードパーティサーバーに送信されることは一切ありません。
